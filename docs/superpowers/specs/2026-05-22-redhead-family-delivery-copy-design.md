# 红头顽家品牌与交付说明优化设计

## 目标

在不改动现有本地 AI、设备模式、本地存储与学习功能逻辑的前提下，把项目品牌统一为“红头顽家 AI 学习助手”，并把家长交付方式拆成电脑 AI 增强版与手机 / 平板学习工具箱版两条清楚路线。

## 品牌范围

- `APP_NAME` 使用 `红头顽家 AI 学习助手｜免费本地版`。
- `ORGANIZATION_NAME` 使用 `红头顽家`。
- 浏览器标题、PWA manifest、Apple Web App 标题、页脚、批处理窗口标题与 README 中的旧品牌同步替换。
- 说明文案里不再出现“舞动成长 AI 学习助手”或“舞动成长本地学习助手”。

## 交付说明

### 电脑 AI 增强版

面向愿意准备 Windows 本地环境的家长，说明：

- 适合有 Windows 电脑、愿意安装 Node.js 与 Ollama、网络能下载 `qwen3:1.7b`、想体验本地 AI 答疑的家庭。
- 机构把项目 zip 压缩包发给家长。
- 家长解压到桌面。
- 第一次双击 `install.bat`，检查或安装 Ollama，并准备 `qwen3:1.7b`。
- 后续双击 `start.bat`，打开 `http://localhost:3000`。

README 的 zip 清单明确分组：

- 保留 `install.bat`、`start.bat`、`README.md`、`package.json`、`package-lock.json`、`src`、`public`、`next.config`、`tsconfig` 与必要配置文件。
- 不打包 `node_modules`、`.next`、`.git`、`dev-server`、`dev-server-error`、`tsconfig.tsbuildinfo`。

### 手机 / 平板学习工具箱版

面向只想直接用手机或平板的家庭，说明：

- 默认使用学习工具箱模式。
- 工具箱包括错题本、学习计划、作文结构模板、数学解题步骤模板、英语作文模板、家长周报模板与每日打卡。
- 手机和平板不强制使用本地 AI。
- 电脑本地 AI 增强版是可选功能，不是必须步骤。

## 手机 / 平板测试说明

README 与家长安装引导页都增加两种说明：

1. 同一 WiFi 测试
   - 电脑先运行 `start.bat`。
   - 手机和平板连同一个 WiFi。
   - 在电脑查看本机 IPv4 地址。
   - 手机访问 `http://电脑IP地址:3000`，示例 `http://192.168.1.8:3000`。
   - 手机打不开时提示 Windows 防火墙可能拦截，需要允许 Node.js 或浏览器开发服务通过。
   - 手机访问的是电脑当前运行中的页面，电脑关机或关闭启动脚本后访问会失效。
2. 正式分享
   - 外地家长或不同 WiFi 家长需要部署网页链接。
   - 手机版网页链接默认不启用电脑本地 AI，只提供学习工具箱、错题本、学习计划、作文模板和家长报告模板。

## 首页入口

首页首屏入口调整为三个面向家长的动作：

- 开始学习，进入现有学习入口。
- 家长安装引导，进入安装引导页。
- 手机/平板使用说明，定位到安装引导页的手机说明区域。

首页继续保持当前产品首屏节奏，不把完整交付文档堆进 hero。

## 测试与边界

- 新增或调整测试，覆盖品牌名、首页三个入口、安装引导页两版交付说明、手机测试说明和 manifest 名称。
- 不调整 AI 调用、localStorage、设备检测、WebLLM、Ollama 模型策略与现有学习页面功能。
- 实现后运行测试、类型检查和构建；如 Codex 终端缺少 `npm`，记录原因并用项目可执行入口完成等价验证。
